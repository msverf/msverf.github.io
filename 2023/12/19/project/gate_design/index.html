题目：使用 Magic 设计 NOR3

# Step 1 设计 nor3

使用 AMS 0.35μm 技术（c35b4）来设计三输入或非门，可在 Linux 命令行使用


来打开 Magic 同时命名文件为 nor3.mag。

根据下面的棍图（stick diagram）来进行门电路设计。注意其中存在的 AMS 0.35μm 所使用的设计规则。其中，pmos 的宽度设定为 12.6μm，要根据仿真来确定 nmos 的宽度。

# Step 2 将 nor3 进行 heirarchy

对 nor3 进行仿真前，需要先进行 heirarchy。在 Magic 中，通过 getcell 命令，可以将三个或非门进行 heirarchy 操作，将该文件命名为 nor3_ld.mag。
门的测量性能将取决于输入驱动条件和输出负载条件。为了获得有用的结果，我们必须确保真实的驱动和负载条件。
对于或非门，将两个负载的其中一个输入接入驱动或非门的输出，另外两个输出接低电平，即 GND。对于与非门，则另外两个输入需要接高电平，即 Vdd。

最终获得经过 hierarchy 后的 NOR3 的版图：

# Step 3 提取仿真文件并修改参数

Magic 中提供了提取并生成仿真所使用的 sv 文件的方法。


对于输入信号，A、B、C 不能同时接入，可依次设置延迟 2ns。根据第一次仿真的结果，修改输入信号的斜率与输出信号大致相同。

要确定合适的 nmos 宽度，要测量输出信号的上升延迟和下降延迟（右图），当两者大致相同时，确定此时的 nmos 宽度。当然也可以进行输入输出信号传播延迟的测量（左图）。

为了测量准确，需要取延迟的平均值，可将 A、B、C 信号分别作为前、中、后输入的信号设置输入参数。
确定好测量方法后，打开 sp 文件修改参数。


为了正确找到合适的 nmos 宽度值，即输出信号的上升延迟与下降延迟大致相等时。我们可以使用 SWEEP 语法来进行一个频率段内的扫描。

相应的，在 spice 文件中，我们需要修改扫描的 extra 值。

# Step 4 仿真，根据结果修改版图

修改完参数后，进行仿真：


获得对应的波形图：

通过 cat 命令可以查看 SWEEP 的结果：


找到 varg_f 和 varg_r 最接近的一组，通过查看 SWEEP 的值，可以确定此时具体的 nmos 宽度的单位。根据该值修改版图 nor3 中 nmos 的宽度，将 SWEEP 的范围修改为 -1 +1 1 ，查看是否在 0 时，上升延迟与下降延迟相差最小。
通过 cat 查看结果：

可以看出，SWEEP 值为 0 时，输出信号的上升延迟与下降延迟最接近。此时 nmos 的宽度为最合适的宽度。
